standards for the wiki

see also 

 page format

  • use markdown for anything other than trivial stuff because that way it can be easily exported to latex
  • only use HTML if you really have to import external content as is

 headings

 heading 1 should be a title, that way stitching multiple notes  into a LaTeX PDF would be painless. 

 links

 for the sake of compatibility with mweb  use the following syntax always:

 

[ display text](./display-text.md)

always use relative links for compatibility with other tools and wikis (e.g. pepperminty, github wiki etc. )

name space

 make all files under a directory, but, avoid deeply nesting things because otherwise there will be lots of link root.
